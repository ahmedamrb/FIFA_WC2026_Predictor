# FIFA WC 2026 Predictor — Agent Instructions

## Project Overview

Python ML system predicting FIFA World Cup 2026 match outcomes and scorelines. Trains on historical international football results and FIFA rankings; serves predictions via a local Streamlit dashboard.

Full spec: [PRD](PRD_FIFA_WC2026_Predictor.md) | Implementation plan: [Plan](FIFA_WC2026_Predictor_Implementation_Plan.md)


### Rankings Merge
For each match row, use the **most recent ranking on or before `match_date`** — not the current ranking. Fill missing rankings with the global median.

## ML Conventions

### Two Parallel Tracks
- **Track A — Outcome model**: Multi-class classifier (0/1/2). Primary: XGBoost. Ensemble: XGB + RF + LR (soft-voting average of probabilities).
- **Track B — Goals model**: Two separate `PoissonRegressor` / `XGBRegressor` models (home goals, away goals).

### Evaluation Metrics (Targets)
| Metric | Target |
|--------|--------|
| Log-loss (val, WC 2022) | < 0.95 |
| Accuracy (val, WC 2022) | > 52% |
| Brier score | < 0.22 |
| Flat-stake bet ROI (WC 2018+2022) | > −10% |

### Tuning
- Use `Optuna` TPE sampler, 100 trials, `StratifiedKFold(n_splits=5)`.
- Primary metric: log-loss. Secondary: accuracy.
- Best params persisted to `data/processed/best_hyperparams.json`.

### Model Serialisation
`joblib.dump` to `models/`; filenames: `outcome_xgb.pkl`, `outcome_rf.pkl`, `outcome_lr.pkl`, `home_goals_xgb.pkl`, `away_goals_xgb.pkl`, `home_goals_poisson.pkl`, `away_goals_poisson.pkl`.

Always record in [MODEL_REGISTRY.md](models/MODEL_REGISTRY.md): filename, type, key hyperparams, val log-loss/MAE, date trained.

## Feature Engineering Key Details

All features built in `src/data/preprocess.py`. The constant `FEATURE_COLUMNS` must stay in sync with the actual columns in both parquet files.

- **Form features**: last 5 and last 10 prior matches — wins, avg goals scored, avg goals conceded, WDL points. Fill nulls with global median.
- **H2H features**: last 5 prior meetings (home/away symmetric). Fill zero-history rows with neutral defaults (win_rate=0.33, avg_goals_home=1.3, avg_goals_away=1.1).
- **Rankings merge**: `rank_diff = home_rank - away_rank`, `rank_points_ratio` capped to [0.1, 10].
- **Context features**: `tournament_stage` ordinal (0=non-WC, 1=Group, 2=R32, 3=R16, 4=QF, 5=SF/3rd, 6=Final), `is_wc_match`, `is_neutral_venue`, `host_nation_advantage`, days-rest (fill first match with 30).

## Anti-Patterns

❌ Don't use current FIFA rankings for historical matches — use the ranking at match date  
❌ Don't include WC 2018 or WC 2022 rows in the training set — they are val/test splits  
❌ Don't fit scalers on val/test data — fit on `X_train` only  
❌ Don't load `.pkl` files from `models/` during EDA or feature engineering scripts  
❌ Don't hardcode team names — use `team_name_map.csv` for cross-dataset alignment  
❌ Don't make external HTTP calls at Streamlit runtime — all data must be pre-fetched  
❌ Don't bypass the GW offset equivalent: form/h2h features must use only data **before** `match_date`  
❌ Don't store `FD_API_KEY` or any secret in source code — read from `.env` via `python-dotenv`
